paso consiste en traducir la parte nombre del servidor del url en una direcci\u00f3n ip usando la base de datos distribuida de internet conocida como dns. esta direcci\u00f3n ip es necesaria para contactar con el servidor web y poder enviarle paquetes de datos.\nel siguiente paso es enviar una petici\u00f3n http al servidor web solicitando el recurso. en el caso de una p\u00e1gina web t\u00edpica, primero se env\u00eda el texto html y luego el mismo es inmediatamente analizado por el navegador, el cual, despu\u00e9s, hace peticiones adicionales para los gr\u00e1ficos y otros ficheros que formen parte de la p\u00e1gina. las estad\u00edsticas de popularidad de un sitio web normalmente est\u00e1n basadas en el n\u00famero de p\u00e1ginas vistas o peticiones de servidor asociadas, o peticiones de fichero, que tienen lugar.\nal recibir los ficheros solicitados desde el servidor web, el navegador representa (renderiza) la p\u00e1gina tal y como se describe en el c\u00f3digo html, el css y otros lenguajes web. al final se incorporan las im\u00e1genes y otros recursos para producir la p\u00e1gina que ve el usuario en su pantalla.\n\n\n== est\u00e1ndares web javascript ==\nun avance significativo en la tecnolog\u00eda web fue la plataforma java de sun microsystems. este lenguaje permite que las p\u00e1ginas web contengan peque\u00f1os programas (llamados applets) directamente en la visualizaci\u00f3n. estos applets se ejecutan en el ordenador del usuario, proporcionando una interfaz de usuario m\u00e1s rico que simples p\u00e1ginas web. los applets java del cliente nunca obtuvieron la popularidad que sun esperaba de ellos, por una serie de razones, incluyendo la falta de integraci\u00f3n con otros contenidos (los applets fueron confinados a peque\u00f1as cajas dentro de la p\u00e1gina renderizada) y el hecho de que muchos ordenadores del momento eran vendidos a los usuarios finales sin una jvm correctamente instalada, por lo que se necesitaba que el usuario descargara la m\u00e1quina virtual antes de que el applet comenzara a aparecer. hasta la llegada de html5, adobe flash desempe\u00f1\u00f3 muchas de las funciones que originalmente se pensaron que podr\u00edan hacer los applets de java incluyendo la ejecuci\u00f3n de contenido de v\u00eddeo, animaciones y algunas caracter\u00edsticas superiores de gui. en estos momentos java se utiliza m\u00e1s como plataforma y lenguaje para el lado del servidor y otro tipo de programaci\u00f3n.\njavascript, en cambio, es un lenguaje de script que inicialmente fue desarrollado para ser usado dentro de las p\u00e1ginas web. la versi\u00f3n estandarizada es el ecmascript. si bien los nombres son similares, javascript fue desarrollado por netscape y no tiene relaci\u00f3n alguna con java, aparte de que sus sintaxis derivan del lenguaje de programaci\u00f3n c. en uni\u00f3n con el document object model de una p\u00e1gina web, javascript se ha convertido en una tecnolog\u00eda mucho m\u00e1s importante de lo que pensaron sus creadores originales. la manipulaci\u00f3n del modelo de objetos de documento despu\u00e9s de que la p\u00e1gina ha sido enviada al cliente se ha denominado html din\u00e1mico (dhtml), para enfatizar un cambio con respecto a las visualizaciones de html est\u00e1tico.\nen su forma m\u00e1s simple, toda la informaci\u00f3n opcional y las acciones disponibles en las p\u00e1ginas web con javascript ya son cargadas la primera vez que se env\u00eda la p\u00e1gina. ajax (\"asynchronous javascript and xml\", en espa\u00f1ol, javascript as\u00edncrono y xml) es una tecnolog\u00eda basada en javascript que puede tener un efecto significativo para el desarrollo de la web. ajax proporciona un m\u00e9todo por el cual grandes o peque\u00f1as partes dentro de una p\u00e1gina web pueden actualizarse, usando nueva informaci\u00f3n obtenida de la red en respuesta a las acciones del usuario. esto permite que la p\u00e1gina sea mucho m\u00e1s confiable, interactiva e interesante, sin que el usuario tenga que esperar a que se cargue toda la p\u00e1gina. ajax es visto como un aspecto importante de lo que suele llamarse web 2.0. ejemplos de t\u00e9cnicas ajax usadas actualmente pueden verse en gmail, google maps, etc.\n\n\n== implicaciones sociol\u00f3gicas ==\nla web, tal y como la conocemos hoy d\u00eda, ha permitido un flujo de comunicaci\u00f3n global a una escala sin precedentes en la historia humana. personas separadas en el tiempo y el espacio, pueden usar la web para intercambiar- o incluso desarrollar mutuamente- sus pensamientos m\u00e1s \u00edntimos, o alternativamente sus actitudes y deseos cotidianos. experiencias emocionales, ideas pol\u00edticas, cultura, idiomas musicales, negocio, arte, fotograf\u00edas, literatura... todo puede ser compartido y diseminado digitalmente con el menor esfuerzo, haci\u00e9ndolo llegar casi de forma inmediata a cualquier otro punto del planeta. aunque la existencia y uso de la web se basa en tecnolog\u00eda material, que tiene a su vez sus propias desventajas, esta informaci\u00f3n no utiliza recursos f\u00edsicos como las bibliotecas o la prensa escrita. sin embargo, la propagaci\u00f3n de informaci\u00f3n a trav\u00e9s de la web (v\u00eda internet) no est\u00e1 limitada por el movimiento de vol\u00famenes f\u00edsicos, o por copias manuales o materiales de informaci\u00f3n. gracias a su car\u00e1cter virtual, la informaci\u00f3n en la web puede ser buscada m\u00e1s f\u00e1cil y eficientemente que en cualquier medio f\u00edsico, y mucho m\u00e1s r\u00e1pido de lo que una persona podr\u00eda recabar por s\u00ed misma a trav\u00e9s de un viaje, correo, tel\u00e9fono, tel\u00e9grafo, o cualquier otro medio de comunicaci\u00f3n.\nla web es el medio de mayor difusi\u00f3n de intercambio personal aparecido en la historia de la humanidad, muy por delante de la imprenta. esta plataforma ha permitido a los usuarios interactuar con muchos m\u00e1s grupos de personas dispersas alrededor del planeta, de lo que es posible con las limitaciones del contacto f\u00edsico o simplemente con las limitaciones de todos los otros medios de comunicaci\u00f3n existentes combinados.\ncomo bien se ha descrito, el alcance de la red hoy d\u00eda es dif\u00edcil de cuantificar. en total, seg\u00fan las estimaciones de 2010, el n\u00famero total de p\u00e1ginas web, bien de acceso directo mediante url, bien mediante el acceso a trav\u00e9s de enlace, es de m\u00e1s de 27 000 millones; es decir, unas tres p\u00e1ginas por cada persona viva en el planeta.[29]\u200b a su vez, la difusi\u00f3n de su contenido es tal, que en poco m\u00e1s de 10 a\u00f1os, hemos codificado medio bill\u00f3n de versiones de nuestra historia colectiva, y la hemos puesto frente a 1900 millones de personas. es en definitiva, la consecuci\u00f3n de una de las mayores ambiciones de la humanidad: desde la antigua mongolia, pasando por la biblioteca de alejandr\u00eda o la mism\u00edsima enciclopedia de rousseau y diderot, la humanidad ha tratado de recopilar en un mismo tiempo y lugar todo el saber acumulado desde sus inicios hasta ese momento. el hipertexto ha hecho posible ese sue\u00f1o.\ncomo la web tiene un \u00e1mbito de influencia global, se ha sugerido su importancia en la contribuci\u00f3n al entendimiento mutuo de las personas por encima de fronteras f\u00edsicas o ideol\u00f3gicas.\n\n\n== publicaci\u00f3n de p\u00e1ginas ==\nla web est\u00e1 disponible como una plataforma m\u00e1s englobada dentro de los mass media. para encontrar el prefijo \"www\" al comienzo de las direcciones web debido a la costumbre de nombrar a los host de internet (los servidores) con los servicios que proporcionan. de esa forma, por ejemplo, el nombre de host para un servidor web normalmente es \"www\", para un servidor ftp se suele usar \"ftp\", y para un servidor de noticias, usenet, \"news\" o \"nntp\" (en relaci\u00f3n con el protocolo de noticias nntp). estos nombres de host aparecen como subdominio de dns, como en \"www.example.com\".\nel uso de estos prefijos no est\u00e1 impuesto por ning\u00fan est\u00e1ndar, de hecho, el primer servidor web se encontraba en \"nxoc01.cern.ch\"[32]\u200b e incluso hoy en d\u00eda existen muchos sitios web que no tienen el prefijo \"www\". este prefijo no tiene ninguna relaci\u00f3n con la forma en que se muestra el sitio web principal. el prefijo \"www\" es simplemente una elecci\u00f3n para el nombre de subdominio del sitio web.\nalgunos navegadores web a\u00f1aden autom\u00e1ticamente \"www.\" al principio, y posiblemente \".com\" al final, en las url que se teclean, si no se encuentra el host sin ellas. internet explorer, mozilla firefox y opera tambi\u00e9n a\u00f1adir\u00e1n \"http://www/.\" y \".com\" al contenido de la barra de direcci\u00f3n si se pulsan al mismo tiempo las teclas de control y \u21b5 entrar (tecla enter). por ejemplo, si se teclea \"ejemplo\" en la barra de direcciones y luego se pulsa solamente \u21b5 entrar o control+\u21b5 entrar normalmente buscar\u00e1 \"http://www.ejemplo.com\", dependiendo de la versi\u00f3n exacta del navegador y su configuraci\u00f3n.\n\n\n== tecnolog\u00edas web ==\nlas tecnolog\u00edas web implican un conjunto de herramientas que nos facilitar\u00e1n lograr mejores resultados a la
