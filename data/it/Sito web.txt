"un sito web (o sito internet oppure detto semplicemente sito) \u00e8 un insieme di pagine web correlate, ovvero una struttura ipertestuale di documenti informativi che risiede, tramite hosting, su un server web, raggiungibile dall'utente finale attraverso un comune motore di ricerca e/o un indirizzo web tramite l'uso di browser web.\n\n\n== descrizione ==\n\n\n=== struttura ===\ndal punto di vista dell'organizzazione o struttura dei contenuti tipicamente un sito web \u00e8 strutturato in una pagina principale (home page), che rappresenta la pagina principale di presentazione del sito e dei suoi contenuti e dalle pagine web secondarie dei contenuti, raggiungibili dai men\u00f9 e widget presenti nell'home page stessa tramite link interni (ancore) al sito stesso. quando il sito web costituisce un punto di partenza, un portone di ingresso, per raggruppare servizi o informazioni web di tipi differenti (di solito sviluppate e gestite in modo autonomo l'una dall'altra), si parla pi\u00f9 propriamente di portale web. spesso inoltre un sito web si accoppia/accompagna ad un database sullo stesso web server per la memorizzazione strutturata di dati e informazioni. l'home page di un sito \u00e8 la pagina che si ottiene digitando il solo nome di dominio.\nper esempio, nell'indirizzo www.w3c.org/consortium/offices/role.html:\n\nwww.w3c.org \u00e8 la radice, o nome di dominio\n/consortium/offices/ sono le sottocartelle, separate dal simbolo \"/\"\nrole.html \u00e8 il nome della paginasolitamente le pagine di un sito risiedono tutte sullo stesso web server, ossia lo spazio fisico in cui risiedono i files che compongono il sito, e la ramificazione in sottocartelle dell'indirizzo corrisponde ad una uguale ramificazione nell'hard disk dello stesso server. ovviamente l'accesso ad un sito o alle pagine web per eventuali modifiche su contenuti e/o forma da parte degli sviluppatori \u00e8 del tutto riservato all'amministratore del sito tramite codici o chiavi di accesso.\nper l'interpretazione del codice che compone una pagina web vengono comunemente usati i browser, ovvero quei software che hanno il compito di interpretare il codice e restituirne un output grafico. tra i browser pi\u00f9 comunemente usati possiamo trovare (a titolo indicativo e non esaustivo) mozilla firefox, google chrome, opera, microsoft internet explorer, microsoft edge (os windows 10) e safari. i file interpretati da questi software all'accesso della pagina web sono solitamente i fogli di stile (css) e i vari script che vengono richiamati all'accesso della pagina. questi determinano dunque l'aspetto grafico finale restituito all'utente all'accesso della pagina.\n\n\n=== tipologia ===\ndal punto di vista della struttura e della tecnologia adoperata, i siti web si possono distinguere sommariamente in due tipi principali:\n\nsiti statici;\nsiti dinamici.a loro volta alcune agenzie web pi\u00f9 professionali tendono a volerli differenziare ancora in:\n\nsiti vetrina - sito di presentazione;\nsiti web dinamico - di solito usato da aziende che necessitano di aggiornamenti indipendenti su parti come blog o applicativi annessi, tipo: calendario, caricamento file mediatici, gestione di richieste a form,...;\npiattaforma e-commerce - usato per la vendita di prodotti e/o servizi aziendali;\ncodice su misura - programmazione avanzata su misura di tecnologie web, tipo: html5, css3, js, jquery. di solito queste tecnologie vengono progettate pensando alla risoluzione di specifici problemi permettendo di ottimizzare al meglio le risorse aziendali.molto importante \u00e8 capire che lo sviluppo web su misura \u00e8 caratteristica assai complessa da elaborare e che i relativi costi di produzione possono variare di molto sulla scala economica. i siti web statici formati da pagine statiche presentano contenuti di sola ed esclusiva lettura, solitamente vengono aggiornati con una bassa frequenza e sono mantenuti da una o pi\u00f9 persone che agiscono direttamente sul codice html della pagina (tramite appositi editor web). si tratta storicamente della prima generazione di siti web.\ni siti web dinamici formati da pagine web dinamiche presentano invece contenuti redatti dinamicamente (in genere grazie al collegamento con un database) e forniscono contenuti che possono variare in base a pi\u00f9 fattori. i siti web dinamici sono caratterizzati da un'alta interazione fra sito e utente; alcuni elementi che caratterizzano la dinamicit\u00e0 di un sito possono essere: l'interazione con uno o pi\u00f9 database, la visualizzazione dell'ora server, operazioni varie sul file system (tipo creazione dinamica di documenti, ecc.), il cloaking basandosi su vari parametri (tra i quali lo user agent, o il browser, o la risoluzione dell'utente), la visualizzazione o stampa o catalogazione degli indirizzi ip degli utenti, e molto altro.\nessi possono essere scritti tramite linguaggi di scripting come asp, php e molti altri linguaggi web. particolarit\u00e0 dei siti dinamici \u00e8 che le \"pagine\" che costituiscono il sito vengono generate all'atto della chiamata da parte dell'utente (fatta eccezione per i sistemi che adottano una cache) in quanto i contenuti non sono memorizzati direttamente nel codice sorgente della pagina, bens\u00ec in sistemi di memorizzazione esterni allo stesso, come le gi\u00e0 citate basi di dati. la modifica dei contenuti, che spesso possono essere aggiornati grazie ad alcuni editor grafici (detti wysiwyg) anche senza agire direttamente sul codice, \u00e8 generalmente frequente. un chiaro esempio di sito dinamico \u00e8 www.wikipedia.org.\n\n\n=== categorie ===\ni siti web vengono poi comunemente divisi in categorie per inquadrarne il settore di operativit\u00e0 o i servizi offerti:\n\nsito personale cv - condotto da individui o piccoli gruppi di persone (come una famiglia), che contiene informazioni prevalentemente autobiografiche o focalizzate sui propri interessi personali (ad es. un blog)\nsito aziendale - funzionale alla promozione di un'azienda o di un servizio\nsito di commercio elettronico (o \"e-commerce\") - specializzato nella vendita di beni e/o servizi via internet\nsito comunit\u00e0 - un sito in cui utenti possono comunicare fra loro, per esempio tramite chat o forum, formando una comunit\u00e0 virtuale\nsito di download - ampie raccolte di link per scaricare software (dimostrativi di videogiochi, immagini, ecc.)\nforum: luogo in cui discutere tramite la pubblicazione e la lettura di messaggi, organizzati per discussioni (thread) e messaggi (post)\nsito informativo - con contenuti tesi a informare l'utente, ma non strettamente per scopi commerciali\nmotore di ricerca - registra i contenuti degli altri siti e li rende disponibili per la ricerca\ndatabase - un sito il cui uso principale \u00e8 di ricercare e mostrare il contenuto di uno specifico database (ad esempio l'internet movie database per i film)\nsito ludico - un sito che \u00e8 in s\u00e9 un gioco oppure serve da arena per permettere a pi\u00f9 persone di giocare\naggregatore di notizie - presenta contenuti prelevandoli automaticamente dalla rete da molte fonti contemporaneamente, ed \u00e8 simile in questo ai motori di ricerca che sfruttano i risultati delle ricerche degli utenti aggregandoli e immagazzinandoli in un database con criteri come il tipo di media (foto, video, testo etc.) e il contenuto testuale. il risultato \u00e8 un sito che cresce arricchendosi di pagine con contenuto inerente all'argomento di base, definito in fase di installazione ma affinato sulle preferenze degli utenti, il tutto in modo automatico.\nlink farm - siti creati unicamente per proporre collegamenti verso altri siti a scopo pubblicitario (spesso in cambio di canoni in denaro)molti siti sono un incrocio tra due o pi\u00f9 delle precedenti categorie. ad esempio, un sito aziendale pu\u00f2 allo stesso tempo pubblicizzare i propri prodotti e pubblicare materiale informativo. prende il nome di portale un sito web che ha assunto dimensioni tali da costituire un punto di partenza, una porta di ingresso ad un gruppo consistente di risorse per gli utenti del web; l'argomento del portale pu\u00f2 essere pi\u00f9 o meno generalista oppure specializzato (portale verticale). taluni siti web presentano aree riservate, ovvero sezioni accessibili solamente previo pagamento di un canone, registrazione, o assegnazione di una password.\n\n\n=== accesso ===\ni siti risiedono, tramite hosting, su un server web e sono accessibili all'utente client che ne inoltri richiesta di accesso tramite un web browser sul world wide web della rete internet, digitando in esso il rispettivo url o direttamente l'indirizzo ip. le pagine di un sito web sono accessibili tramite una radice comune (detta \"nome di dominio\", per esempio \"www.kernel.org\"), seguita da una serie opzionale di \"sotto cartelle\" e dal nome della pagina. il nome completo di ogni pagina \u00e8 detto \"indirizzo web\" o, pi\u00f9 tecnicamente, uri (o url).\n\n\n== progettazione, realizzazione e gestione ==\nla progettazione di un sito web \u00e8 analoga a larghe linee al modello di sviluppo del software tipica dell'ingegneria del software con la definizione/pianificazione iniziale di ci\u00f2 che deve essere realizzato a livello logico (analisi/definizione delle specifiche, strutturazione dei contenuti e grafica/formattazione), seguita dalla fase implementativa e dalla fase di testing e successiva eventuale correzione delle incongruenze con le specifiche o l'usabilit\u00e0 del sito stesso. la realizzazione/gestione di un sito web fa ampio uso di tecniche e conoscenze proprie della programmazione web o di format predefiniti direttamente a disposizione dell'utente amministratore forniti da strumenti appositi o reperibili in rete e dall'uso tipicamente user friendly.\n\n\n=== i linguaggi ===\nil linguaggio pi\u00f9 diffuso con cui i siti web sono realizzati \u00e8 l'html (hyper text markup language), i suoi derivati e i css. in particolare l'html viene interpretato da particolari software chiamati web browser. alcuni plugin per i browser, come flash, shockwave o applet java, permettono la visualizzazione di contenuti animati. per i siti web dinamici dove alcuni contenuti possono essere generati dinamicamente sul browser dell'utente, in aggiunta ai precedenti linguaggi si usano linguaggi di scripting come javascript o dynamic html, tecnologie supportate per impostazione predefinita da tutti i browser recenti (programmazione web client-side).\nper la costruzione/realizzazione di siti web dinamici in grado invece di estrapolare dati da database, inviare email, gestire informazioni, ecc., i linguaggi di scripting pi\u00f9 diffusi sono php, ruby, python e asp in alternativa al java, solitamente usato per applicazioni enterprise (programmazione web server-side). per la creazione di siti internet molti webmaster usano comunemente strumenti automatizzati, chiamati webeditor (es. wysiwyg), grazie ai quali possono redigere il codice anche senza avere un'approfondita conoscenza dei linguaggi. sempre pi\u00f9 la realizzazione e la gestione di siti web complessi e ricchi di contenuti passa attraverso l'uso di content management system (cms), delle vere piattaforme web (sorta di evoluzione dei web editor) per le quali non \u00e8 richiesta alcuna conoscenza di tecniche di programmazione web ed utili anche per forum, blog e portali di e-commerce. queste piattaforme spesso si appoggiano su piattaforme xampp.\n\n\n=== accessibilit\u00e0 ===\ni siti web devono soddisfare dei requisiti di accessibilit\u00e0 che rendano i suoi contenuti fruibili a chiunque, compresi i soggetti affetti da disabilit\u00e0 psichica e fisica, definiti in regole precise denominate web content accessibility guidelines (wcag) definite dal world wide web consortium (w3c).\n\n\n=== figure professionali ===\ntra le figure professionali legate alla creazione dei siti web vi sono:\n\nl'esperto di architettura dell'informazione (information architect), che \u00e8 responsabile dell'organizzazione dell'intero sito e della struttura di navigazione.\nil webdesigner, che progetta il sito dall'aspetto grafico (layout) ai contenuti.\nil web manager, che coordina il progetto web del sito da realizzare;\nil content manager, che \u00e8 responsabile della redazione del contenuto e della struttura logica.\nil web developer o sviluppatore, che \u00e8 responsabile del comportamento delle pagine.\nil responsabile seo (search engine optimization), che si occupa del posizionamento del sito all'interno degli indici dei motori di ricerca.\nl'esperto di webmarketing, che si occupa della promozione del sito nel web.\nil web strategist, che si occupa di gestire la presenza sul web delle aziende.\nil webmaster, che amministra il sito in generale, dunque ne ha accesso esclusivo tramite relativi codici o chiavi di accesso, e ne \u00e8 il responsabile, anche della pubblicazione (messa on line), revisione/aggiornamento nonch\u00e9 degli aspetti legali. mantiene i rapporti con il committente/proprietario del sito web.\nil sistemista, che gestisce la rete, l'hardware e il software di base del web server in cui \u00e8 ospitato il sito (cio\u00e8 l'infrastruttura che ospita il sito). il sistemista \u00e8 l'unico professionista che non si occupa del sito web in termini propri ma solo del supporto di base (sistemi operativi, configurazione database e applicativi, architettura e logiche hd, prestazioni, ecc).nel panorama delle professionalit\u00e0 di rete si \u00e8 cercato di dare una regolamentazione basandosi sui modelli europei e-cf con la creazione da parte di iwa - entit\u00e0 di standardizzazione per le professioni web - di una serie di figure professionali secondo criteri di competenze e conoscenze.\n\n\n== vulnerabilit\u00e0 dei siti web ==\n\n\n=== siti dinamici ===\nsebbene esistano tecniche per aumentare la sicurezza, nei siti dinamici potrebbero verificarsi:\n\nbackdoor: metodo per eludere la procedura d'accesso al server del sito web\nsql injection o sqli: un hacker pu\u00f2 inserire istruzioni sql dannose nel sito ottenendo potenzialmente l\u2019accesso a dati sensibili presenti nel database o distruggendo questi dati\npharma hack: l'hacker distribuisce pubblicit\u00e0 farmaceutiche ai visitatori attraverso link incorporati e anchor text sulle pagine del sito di un altro utente\nredirect dannosi: l'hacker pu\u00f2 nascondere degli script sul sito di un altro utente e renderizzare i visitatori da altre parti\ndownload drive-by: l'hacker pu\u00f2 inserire degli script sul sito di un altro utente per fare in modo che i visitatori scarichino software a loro insaputa\ncross-site scripting (xss): permette a un cracker di inserire o eseguire codice lato client al fine di attuare un insieme variegato di attacchi quali, ad esempio, raccolta, manipolazione e reindirizzamento di informazioni riservate, visualizzazione e modifica di dati presenti sui server, alterazione del comportamento dinamico delle pagine web, ecc.\nfile inclusion: local file inclusion (lfi) consente a un utente malintenzionato di includere file su un server tramite il browser web. remote file inclusion, o rfi, nell'ambito della sicurezza informatica indica una vulnerabilit\u00e0 che affligge i servizi web con uno scarso controllo delle variabili arrivate da un utente, in particolare le variabili get e post del php.\nil cross-site request forgery, abbreviato csrf o anche xsrf, \u00e8 una vulnerabilit\u00e0 a cui sono esposti i siti web dinamici quando sono progettati per ricevere richieste da un client senza meccanismi per controllare se la richiesta sia stata inviata intenzionalmente oppure no.\nesecuzione di codice in modalit\u00e0 remota: ottenere l'accesso a un account con privilegi riservati almeno al livelloauthor su un sito di destinazione per eseguire codice php arbitrario sul server sottostante, portando a un'acquisizione remota completa.\nprivilege escalation: l'escalation dei privilegi si verifica quando a un utente con privilegi amministrativi inferiori vengono concessi privilegi pi\u00f9 elevati.\nclickjacking: tecnica dannosa che induce gli utenti a fare clic su qualcosa di diverso da quello che si aspettano.\n\n\n=== siti statici ===\nfurto della password di accesso. l'hacker pu\u00f2 quindi cancellare o modificare dall'ftp il sito web e gli altri file presenti (ci\u00f2 pu\u00f2 succedere anche nei siti dinamici)\n\n\n== performance ==\nle cause principali per un sito web lento sono:\n\nhosting: quando il server di web hosting non \u00e8 configurato correttamente pu\u00f2 danneggiare la velocit\u00e0 del sito web\nconfigurazione: se il sito non usa pagine memorizzate nella cache sovraccaricher\u00e0 il server, causando la lentezza o il blocco del sito web\nimmagini pesanti: a una risoluzione alta (adatta per la stampa ma non per il web) e/o una dimensione ampia e/o mancato uso degli sprite css per evitare molte chiamate al server\nriproduzione automatica dei video all'arrivo dell'utente e video ospitati sul proprio server anzich\u00e9 incorporato da youtube, vimeo o altri servizi esterni\nplugin non validi e/o non aggiornati: se si usano plugin mal codificati e/o obsoleti il sito web pu\u00f2 rallentare notevolmente (per i siti dinamici)\nscript esterni: annunci, caricatori di caratteri, ecc. possono avere un enorme impatto sulle prestazioni del sito web. ad esempio usare troppi font incorporati con google fonts o altri plugin di incorporazione font pu\u00f2 rallentare il sito\ncache disattivata\nassenza di un content delivery network (cdn)\n\n\n== premi ==\nannualmente e mensilmente alcune associazioni assegnano premi ai migliori siti web (per quanto riguarda il coding, il design, la creativit\u00e0, l'originalit\u00e0 eccetera).\n\n\n== aspetti giuridici ==\ngli aspetti giuridici relativi alla pubblicazione di documenti su un sito web sono spesso controversi, poich\u00e9 documenti siti in una determinata nazione sono resi accessibili anche da luoghi con leggi ed usi del tutto differenti, ma anche perch\u00e9 le stesse leggi delle singole nazioni seguono con fatica la continua e rapida evoluzione del web. in linea generale i contenuti di un sito sono sottoposti alle condizioni di uso del fornitore dello spazio web (hosting) e alle leggi della nazione in cui esso \u00e8 situato (ad esempio leggi sul copyright, sulla pornografia, ecc.). alcuni ordinamenti prevedono speciali adempimenti a carico dei gestori di siti web. nei paesi germanofoni, ad esempio, \u00e8 obbligatorio inserire nel sito una dichiarazione di propriet\u00e0 e paternit\u00e0 detta impressum.\n\n\n=== in italia ===\nin italia, il dlgs 196/2003 (legge sulla privacy) impone ai siti che raccolgono informazioni personali degli utenti, o svolgono attivit\u00e0 di commercio elettronico, di fornire all'interessato una idonea informativa sul trattamento dei dati personali. tuttavia, nel 2014 una ricerca di federprivacy ha evidenziato che il 67% dei siti italiani violavano tale prescrizione, soggetta a sanzioni dai 6.000 fino ai 36.000 euro.l\u2019art. 2250 del codice civile, modificato dall\u2019art. 42, l. 88/2009 impone alle societ\u00e0 di capitali di pubblicare informazioni legali nei propri atti, nella corrispondenza, nel sito web aziendale e negli altri luoghi virtuali di comunicazione, come email e profili sui social networks. questi dati includono: ragione sociale, sede legale, codice fiscale e partita iva, posta elettronica certificata (pec), ufficio del registro dove si \u00e8 iscritti, numero repertorio economico amministrativo (rea), e altre ancora, in funzione dello specifico assetto societario.\n\n\n== vantaggi e svantaggi dei diversi tipi di siti web ==\n\n\n== tipologia dei siti web ==\n\n\n== registrazione ==\nla registrazione \u00e8 il servizio di creazione, acquisto e mantenimento, su internet, del nome di dominio relativo ad un sito web. viene eseguita da organizzazioni denominate registrar (per l'italia register.it) che agiscono secondo le regole icann. tramite tale procedura il proprietario del nome del sito web pu\u00f2 vantare i relativi diritti.\nl'auth code (abbreviazione di authorization code) \u00e8 la stringa alfanumerica, fornita dal registrar all'utente che ha eseguito la richiesta di registrazione (spesso tramite una web agency o un provider), che dimostra la relativa propriet\u00e0. questa password (insieme alle credenziali di accesso al pannello di amministrazione del sito) \u00e8 fondamentale nel caso di trasferimento del sito web tra vari fornitori di hosting.\n\n\n== note
