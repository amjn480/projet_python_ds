(computadora, laptop, tel\u00e9fono inteligente)\u2014 conectada a la red, que utilice el protocolo de internet o que corresponda al nivel de red del modelo tcp/ip.[1]\u200b en principio se usa en la red global, aunque tambi\u00e9n para aplicaciones locales (como la identificaci\u00f3n de un router en una red cerrada).\nuna direcci\u00f3n ip tiene dos funciones principales: identificaci\u00f3n de la interfaz de red y direccionamiento para su ubicaci\u00f3n.\nla direcci\u00f3n ip no debe confundirse con la direcci\u00f3n mac, que es un identificador de 48 bits expresado en c\u00f3digo hexadecimal, para identificar de forma \u00fanica la tarjeta de red y no depende del protocolo de conexi\u00f3n utilizado en la red.\nla direcci\u00f3n ip puede cambiar a menudo debido a cambios en la red, o porque el dispositivo encargado dentro de la red de asignar las direcciones ip, decida asignar otra ip (por ejemplo, con el protocolo dhcp). a esta forma de asignaci\u00f3n de direcci\u00f3n ip se le denomina tambi\u00e9n direcci\u00f3n ip din\u00e1mica (normalmente abreviado como ip din\u00e1mica).[1]\u200b los sitios de internet que por su naturaleza necesitan estar permanentemente conectados, generalmente tienen la necesidad de una direcci\u00f3n ip fija (com\u00fanmente, ip fija o ip est\u00e1tica). esta no cambia con el tiempo. los servidores de correo, dns, ftp p\u00fablicos y servidores de p\u00e1ginas web necesariamente deben contar con una direcci\u00f3n ip fija o est\u00e1tica, ya que de esta forma se permite su localizaci\u00f3n en la red.[1]\u200b\nlos dispositivos se conectan entre s\u00ed mediante sus respectivas direcciones ip. sin embargo, para las personas es m\u00e1s f\u00e1cil recordar un nombre de dominio que los n\u00fameros de la direcci\u00f3n ip. los servidores de nombres de dominio dns, \"traducen\" el nombre de dominio en una direcci\u00f3n ip. si la direcci\u00f3n ip din\u00e1mica cambia, es suficiente actualizar la informaci\u00f3n en el servidor dns. el resto de las personas seguir\u00e1n accediendo al dispositivo por el nombre de dominio.[2]\u200b\n\n\n=== direcciones ip ===\nlas direcciones ipv4 se expresan mediante un n\u00famero binario de 32 bits permitiendo un espacio de direcciones de hasta aproximadamente 4.294.967.296 direcciones posibles.[3]\u200b[4]\u200b\nlas direcciones ip se pueden expresar como n\u00fameros de notaci\u00f3n decimal: se dividen los 32 bits de la direcci\u00f3n en cuatro octetos. el valor decimal de cada octeto est\u00e1 comprendido en el intervalo de 0 a 255 [el n\u00famero binario de 8 bits 192.168.255.254/18en las primeras etapas del desarrollo del protocolo de internet,[5]\u200b los administradores de internet interpretaban las direcciones ip en dos partes, los primeros 8 bits para designar la direcci\u00f3n de red y el resto para individualizar la computadora dentro de la red. este m\u00e9todo pronto prob\u00f3 ser inadecuado, cuando se comenzaron a agregar nuevas redes a las ya asignadas. en 1981 el direccionamiento internet fue revisado y se introdujo la arquitectura de clases. (classful network architecture).[6]\u200b en esta arquitectura hay tres clases de direcciones ip que una organizaci\u00f3n puede recibir de parte de la internet corporation for assigned names and numbers (icann): clase a, clase b y clase c.[7]\u200b\n\nen una red de clase a, se asigna el primer octeto para identificar la red, reservando los tres \u00faltimos octetos (24 bits) para que sean asignados a los hosts,[8]\u200b de modo que la cantidad m\u00e1xima de hosts es 224 - 2 (se excluyen la direcci\u00f3n reservada para broadcast (\u00faltimos octetos a 1) y de red (\u00faltimos octetos a 0)), es decir, 16 777 214 hosts.\nen una red de clase b, se asignan los dos primeros octetos para identificar la red, reservando los dos octetos finales (16 bits) para que sean asignados a los hosts,[8]\u200b de modo que la cantidad m\u00e1xima de hosts por cada red es 216 - 2, o 65 534 hosts.\nen una red de clase c, se asignan los tres primeros octetos para identificar la red, enviar paquetes a todos los hosts de la red en la que se ubica. se denomina direcci\u00f3n de broadcast.\n(**) la direcci\u00f3n 0.0.0.0 es reservada por la iana para identificaci\u00f3n local.\n(\u2020) las direcciones 127.x.x.x se reservan para designar la propia m\u00e1quina. se denomina direcci\u00f3n de bucle local o loopback.\n(\u2021) la primera direcci\u00f3n se reserva para identificar la red (p.ej. 18.0.0.0), mientras que la \u00faltima direcci\u00f3n se emplea como direcci\u00f3n de difusi\u00f3n o broadcast (p.ej. 18.255.255.255). ese es el motivo por el que el n\u00famero m\u00e1ximo de hosts en una red es siempre igual al n\u00famero de direcciones disponibles en un rango espec\u00edfico menos dos.el dise\u00f1o de redes de clases (classful) sirvi\u00f3 direcciones de clases fue reemplazado por una arquitectura de redes sin clases classless inter-domain routing (cidr)[9]\u200b en el a\u00f1o 1993. cidr est\u00e1 basada en redes de longitud de m\u00e1s fina y granulada, calculando las direcciones necesarias y \"desperdiciando\" las m\u00ednimas posibles.\n\n\n=== direcciones privadas ===\nexisten ciertas direcciones en cada clase de subredes aut\u00f3nomas separadas. un ejemplo de uso es cuando necesitamos agrupar todos los empleados pertenecientes a un departamento de una empresa. en este caso crear\u00edamos una subred que englobara las direcciones ip de estos. para conseguirlo hay que reservar bits del campo host para identificar la subred estableciendo a uno los bits de red-subred en la m\u00e1scara. por ejemplo la direcci\u00f3n 173.17.1.1 con m\u00e1scara 255.255.255.0 nos indica que los dos primeros octetos identifican la red (por ser una direcci\u00f3n de clase b), el tercer octeto identifica la subred (a 1 la subred (campo host a 0) y la direcci\u00f3n para realizar broadcast en la subred (todos los bits del campo host en 1).\nlas redes se pueden dividir en redes m\u00e1s peque\u00f1as para un mejor subredes, se comienza pidiendo \u201cprestados\u201d bits a la parte de host de una direcci\u00f3n dada, dependiendo de la cantidad de subredes que se deseen crear, as\u00ed como del n\u00famero de hosts necesarios en cada subred.\n\n\n=== ip din\u00e1mica ===\nuna direcci\u00f3n ip din\u00e1mica es una ip asignada al usuario, mediante un servidor dhcp (dynamic host configuration protocol). la ip que se obtiene tiene una duraci\u00f3n m\u00e1xima determinada. el servidor dhcp provee par\u00e1metros de configuraci\u00f3n espec\u00edficos para cada cliente que desee participar en la red ip. definici\u00f3n de dhcp (marzo de 1997). dhcp sustituye al protocolo bootp, que es m\u00e1s antiguo. debido a la compatibilidad retroactiva de dhcp, muy pocas redes contin\u00faan usando bootp puro.\nlas ip din\u00e1micas son las que actualmente ofrecen la mayor\u00eda de operadores. el servidor del servicio dhcp puede ser configurado para que renueve las direcciones asignadas cada tiempo determinado.\n\n\n==== ventajas ====\nreduce los costos de operaci\u00f3n a los proveedores de servicios de internet (isp).\nreduce la cantidad de ip asignadas (de forma fija) inactivas.\nel usuario manual,[nota 1]\u200b o por el servidor de la red,[nota 2]\u200b con base en la direcci\u00f3n mac del cliente. muchas personas asocian ip fija con ip p\u00fablica e ip din\u00e1mica con ip necesariamente se desea que la ip no cambie. por eso la ip p\u00fablica se la configura, habitualmente, de manera fija y no din\u00e1mica.\nen el caso de la ip privada es, generalmente, din\u00e1mica y est\u00e1 asignada por un servidor dhcp, pero en algunos casos se configura ip privada fija para poder controlar el acceso a internet o a la red local, otorgando ciertos privilegios dependiendo del n\u00famero de ip que tenemos. si esta cambiara (si se asignase de manera fuera din\u00e1mica) ser\u00eda m\u00e1s complicado controlar estos privilegios (pero no imposible).\n\n\n== direcciones ipv6
